# Gemini Project Instructions

## 응답 지침
- 모든 AI 응답(설명, 요약, 진행 보고, 결과 보고, 산출물 등)은 **한국어**로 작성합니다.
- 기술적인 용어는 필요한 경우 영어를 병용할 수 있으나, 전체적인 문맥은 한국어를 유지합니다.

## 커밋 메시지 규칙
- 커밋 메시지 형식: `type(scope): 본문`
- **type** (영어 사용):
  - `feat`: 새로운 기능 추가
  - `fix`: 버그 수정
  - `docs`: 문서 수정
  - `refactor`: 코드 리팩토링
  - `test`: 테스트 코드 추가/수정
  - `chore`: 빌드 업무 수정, 패키지 매니저 설정 등
- **scope** (영어 사용):
  - `const`: 상수 및 설정
  - `spec`: 명세서
  - `plan`: 계획
  - `task`: 개별 작업
  - `impl`: 구현
- **본문** (한국어 사용): 변경 사항에 대한 핵심 내용을 한국어로 명확하게 작성합니다.

## 명명 규칙
- 브랜치명, 파일명, 폴더명, 코드 내 식별자(변수, 함수, 클래스 등)는 **영어**를 사용합니다.

## 외부 도구 활용
- 최신 라이브러리 정보 및 기술 스택 적용을 위해 반드시 **Context7 MCP Server**를 활용합니다.
